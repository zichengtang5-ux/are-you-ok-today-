# PRD：今天还好 — 独居守护 iOS App Phase 1 MVP

> 版本：v2.0
> 创建日期：2026-06-23
> 状态：待评审
> 产品经理：小产
> 重大变更：平台从微信小程序调整为 iOS 原生 App

---

## 1. 概述

**今天还好**是一款面向独居人士的 iOS App，通过每日温暖的平安确认 + 超时自动通知紧急联系人，构建轻量化的独居安全守护网络。Phase 1 MVP 以 iOS 原生 App 形态上线，充分利用本地推送通知、后台任务、通知交互等系统能力，确保核心守护链路可靠运行。

**核心价值主张**：不叫"签到"，不叫"打卡"——每天问一句"今天还好吗？"，一次点击告诉关心你的人"我没事"。

**平台选择理由**：微信小程序的订阅消息机制无法满足每日定时推送需求（一次性订阅 = 每次授权仅发一条，长期订阅不对本品类开放）。iOS 原生 App 提供本地通知（无频次限制）、通知内快捷回复（1 次点击）、后台任务、HealthKit 接入能力，是核心守护链路唯一可靠载体。

## 2. 背景与证据

### 2.1 市场背景

- 中国独居人口约 **1.2 亿**，且持续增长
- 独居者面临的核心焦虑："如果我出事了，多久才会有人知道？"
- 独居老人/慢性病患者的安全问题是家庭和社会痛点

### 2.2 竞品空白

| 竞品痛点 | 证据来源 |
|----------|----------|
| 通知渠道单一（仅邮件） | 死了么 App 评测 |
| 签到周期不合理（3天才触发） | 死了么 App 功能分析 |
| 无穿戴设备联动 | 全品类竞品扫描 |
| 缺乏主动预警 | 竞品功能对比 |
| 体验粗糙，缺乏信任感设计 | 独居签到/活着么 UI 分析 |

### 2.3 产品机会

- **温暖交互**：以"今天还好"替代"签到"，降低心理负担
- **可靠推送**：iOS 本地通知无频次限制，确保每日守护不中断
- **通知内快捷回复**：无需打开 App，推送横幅一键确认
- **分级通知**：免费/付费分通道，短信成本可控
- **子女生态**：子女付费为父母开通，情感驱动付费
- **Phase 2 演进**：Apple Watch 心率联动 + 分级响应链（自动拨打120）

### 2.4 微信小程序方案否决原因

| 能力 | 微信小程序 | iOS App | 对核心功能的影响 |
|------|-----------|---------|-----------------|
| 每日定时推送 | 订阅消息：一次性授权=1次发送，长期订阅不可申请 | 本地通知：无限制，精确到分钟 | **致命**：小程序无法实现每日自动推送 |
| 推送内快捷回复 | 不支持，需跳转小程序 | Notification Actions，1次点击 | **严重**：增加操作成本 |
| 后台运行 | 不支持 | Background Tasks | **严重**：超时检测不可靠 |
| Apple Watch 联动 | 不支持 | HealthKit | Phase 2 阻塞 |
| 拨打电话 | 不支持 | 可以 | Phase 2 阻塞 |
| 后台定位 | 不支持 | 可以（授权后） | 告警时无法提供地址 |

**结论**：小程序的通知限制直接阻断核心链路，Phase 1 必须以 iOS App 为主体。

## 4. 用户与场景

### 4.1 目标用户

| 用户角色 | 描述 | 核心需求 |
|----------|------|----------|
| **独居者本人** | 独居青年/中年/老人 | 出事时有人知道；心理安全感 |
| **紧急联系人** | 家人、朋友、邻居 | 被守护者出事时能被及时通知 |
| **子女** | 远在他城的子女 | 远程守护独居父母；主动查看安全状态 |

### 4.2 核心场景

**场景 A：独居者日常守护**
> 小李，28岁，独居。每天晚上 8 点，iPhone 弹出通知"今天还好吗？"，她在通知横幅上直接点"今天还好 ✓"，看到"收到，安心了"的反馈。整个过程不到 5 秒，无需打开 App。如果某天忘了回复，系统先温和提醒，超时后自动发短信通知她妈妈。

**场景 B：超时告警与联系人接管**
> 小李今天没回复。10:30（窗口截止+30分钟宽限后），系统给妈妈发短信："小李今天没有回复平安，最后回复时间：昨天 20:15，请及时联系确认。"妈妈打电话确认小李安全后，打开 App 点"已联系，TA没事 ✓"，告警解除。

**场景 C：子女远程守护**
> 小张在深圳工作，妈妈在老家独居。小张在 App 内"为家人开通守护"，通过 iOS 分享功能发送邀请给妈妈。妈妈下载 App 后一键绑定。妈妈不常回复，但不触发告警——系统标记"未确认"，小张收到"妈妈今天还没回复"，小张打电话确认后在 App 代确认"TA没事"。小张付费解锁关怀看板，随时看妈妈的平安日历。

**场景 D：需要帮助**
> 联系人收到告警短信后联系不上被守护者，在 App 点"联系不上，需要帮助"，系统显示建议行动清单：拨打用户电话、拨打 120（提供用户地址）、联系其他紧急联系人。Phase 1 不自动拨打 120，由联系人手动操作。

**场景 E：主动求助**
> 小李突然感觉身体不适，打开 App 首页点击"🚨 紧急求助"，立即看到紧急联系人电话列表（一键拨打）和 120 拨打入口，同时显示自己的地址信息方便告知救援人员。

**场景 F：暂停守护**
> 小李要出差一周，在设置里暂停守护 7 天。暂停期间不发送每日提醒，不触发超时告警。到期后自动恢复，推送"欢迎回来，守护已恢复"。

### 5.3 假设

1. iOS 本地通知权限授权率可通过引导优化达到 ≥ 60%
2. 用户接受从 App Store 下载独立 App（而非小程序即用即走）
3. 用户愿意为"多联系人 + 语音电话通知 + 关怀看板"付费 0.9 元/月
4. 子女付费意愿强于独居者本人付费
5. 服务端定时器能满足超时检测的精度和可靠性
6. Apple IAP 订阅审核能顺利通过

### 5.4 依赖

| 依赖项 | 风险 | 缓解 |
|--------|------|------|
| iOS 通知权限授权率 | 用户可能拒绝，导致无法推送 | 引导流程强调不授权=无守护；App 内备用提醒入口 |
| SMS 服务商稳定性 | 短信发送失败 | 选择主流服务商 + 重试机制 + 付费用户语音电话兜底 |

| Apple 30% 佣金 | 压低实际收入 | 0.9 元定价下短信成本可覆盖，规模化后转 15%（小企业计划） |
| App Store 获客成本 | 独立 App 获客比小程序难 | 内容营销 + 子女社交传播 + ASO 优化 |

## 6. 用户流程

### 6.1 首次使用流程（独居者本人）

```
从 App Store 下载安装
  │
  ▼
打开 App → 输入手机号 + 验证码登录
  │
  ▼
阅读并同意用户协议 + 隐私政策
  ├─ 核心免责条款单独高亮展示（不藏在长文中）
  └─ 必须勾选同意后才能继续
  │
  ▼
填写基本信息（昵称、手机号、当前住址）
  ├─ 昵称：联系人通知中显示身份
  ├─ 手机号：已登录，用于身份识别
  └─ 当前住址：紧急时刻联系人和 120 快速定位
  │
  ▼
设置紧急联系人（至少1人）
  ├─ 输入联系人姓名、手机号、关系
  ├─ 发送短信验证码
  └─ 验证通过 → 保存
  │
  ▼
设置提醒时间窗口（默认：晚上 8-10 点）
  │
  ▼
授权通知权限（iOS 系统弹窗）
  ├─ 授权 → "太好了！每天晚上 8 点会收到'今天还好吗？'"
  └─ 拒绝 → "不授权将无法收到每日'今天还好吗？'提醒，守护功能将无法正常工作"
      └─ 弹出二次确认："确定不授权吗？"，提供"去授权"和"仍然不授权"选项
  │
  ▼
进入守护首页
  ├─ 已授权通知：显示"守护已开启" + 提醒时间摘要 + 联系人摘要
  │   └─ 通知授权状态横幅："🔔 消息推送已授权，每天会准时收到提醒"
  └─ 未授权通知：显示"守护已开启" + "设置已保存完成"
      └─ 警告横幅："⚠️ 你尚未授权消息推送，请在设置中开启，否则无法收到每日提醒"
      └─ 提供"进入首页"和"返回授权"两个选项
```

**引导流程步骤**（5 个步骤页，登录和完成页不计入步骤进度）：
1. 协议页 — 阅读并同意（4 个勾选项：3 条免责条款 + 用户协议/隐私政策）
2. 基本信息 — 昵称、手机号、当前住址
3. 紧急联系人 — 至少添加 1 人
4. 提醒时间 — 设置每日提醒窗口（默认 20:00-22:00）
5. 通知授权 — 引导开启 iOS 通知权限（含通知样式预览）

### 6.2 每日提醒流程

```
到达提醒时间窗口起始时间
  │
  ▼
iOS 本地推送通知："今天还好吗？"
  │
  ├─ 用户在通知横幅点"今天还好 ✓"（Notification Action）
  │   └─ 状态更新为"已回复"（无需打开 App）
  │   └─ 下次打开 App 时显示温暖反馈：
  │       标题："收到，安心了" / 副标题："已收到你的平安"
  │       + "本月平安 X/Y 天"
  │   └─ 预告："明天晚上 8 点再见"
  │
  ├─ 用户打开 App → 点击"今天还好 ✓" → 同上
  │
  └─ 用户未回复
      │
      ▼
  时间窗口结束 → 进入 30 分钟宽限
      │
      ▼
  推送关心式提醒："没收到你的回应，有点担心，看到消息回一下？"
      │
      ├─ 用户回复（通知 Action 或 App 内）→ 状态正常，不触发联系人通知
      │
      └─ 仍未回复 → 30 分钟宽限结束
          │
          ▼
      状态变为"告警" → 通知紧急联系人
          ├─ 免费用户：SMS 短信
          └─ 付费用户：SMS 短信 + 语音电话
```

**技术实现**：
- 本地通知（UNUserNotificationCenter）：设定每日定时触发，无需服务器
- Notification Actions：通知横幅长按/下拉显示"今天还好 ✓"按钮
- 服务端定时器：窗口截止 +30 分钟后检查数据库回复状态
- 关心式提醒：服务端触发远程推送（APNs）

### 6.3 联系人通知与接管流程

```
服务端检测到超时未回复
  │
  ▼
发送 SMS 短信给紧急联系人（按优先级顺序）
  ├─ 免费用户：仅 SMS
  └─ 付费用户：SMS + 语音电话（自动播报告警内容）
  │
  ▼
通知内容：
  "【今天还好】[用户昵称] 今天没有回复平安，
   最后回复时间：XXX，请及时联系确认。
   打开 App 处理：[深链接]"
  │
  ▼
联系人操作：
  ├─ 直接拨打用户电话确认安全（无需 App）
  │
  ├─ 打开 App → 点击"已联系，TA没事 ✓"
  │   └─ 告警解除，记录处理时间
  │   └─ 通知用户："你的联系人已确认你安全"
  │
  └─ 打开 App → 点击"联系不上，需要帮助"
      └─ 显示建议行动清单：
          ├─ 拨打用户电话 [一键拨号]
          ├─ 拨打 120 [一键拨号 + 显示用户地址]
          └─ 联系其他紧急联系人 [列表]
  │
  10 分钟内所有联系人均未确认
      └─ 发送第二轮 SMS 提醒（付费用户同时语音电话）
```

**联系人无需安装 App 也能做的事**：
- 收到 SMS 短信通知
- 直接拨打用户电话确认

**联系人需要 App 才能做的事**：
- 点击"已联系，TA没事"确认告警
- 点击"需要帮助"查看行动清单
- 查看关怀看板（付费用户）

### 6.4 子女端流程

```
子女打开 App → 选择"为家人开通守护"
  │
  ▼
填写父母信息（姓名、手机号）
  │
  ▼
通过 iOS 分享功能发送邀请（iMessage/SMS/微信/其他）
  ├─ 邀请包含唯一邀请码 + App Store 下载链接
  └─ 父母收到短信/消息 → 下载安装 → 输入邀请码或点击深链接自动绑定
  │
  ▼
父母完成绑定后：
  ├─ 子女自动成为第一紧急联系人
  ├─ 子女可代设提醒时间窗口
  └─ 子女可管理父母的其他联系人
  │
  ▼
日常：父母未回复 → 标记"未确认"（非告警）
  ├─ 通知子女："妈妈今天还没回复"（App 推送）
  └─ 子女可代确认"TA没事"
  │
  ▼
关怀看板（付费解锁）：
  ├─ 7天平安日历
  ├─ 连续未回复天数
  └─ 当前状态
```

### 6.5 紧急求助流程

```
用户感到不适/遇到危险
  │
  ▼
打开 App → 点击首页"🚨 紧急求助"入口（所有首页状态均可见）
  │
  ▼
紧急求助页面：
  ├─ 一键拨打 120 [tel:// 拨号]
  ├─ 显示当前地址（基于最后一次 GPS 定位 或 用户预设的当前住址）
  ├─ 紧急联系人列表 [一键拨号]
  └─ 一键通知所有联系人"我需要帮助"（SMS）
  │
  ▼
联系人收到"我需要帮助"短信
```

### R3：每日"今天还好"提醒（REQ-003）

**用户故事**：作为独居者，我希望每天收到一条像朋友问候一样的提醒，点一下就能告诉关心我的人我没事。

**功能行为**：
- 用户设定每日提醒时间窗口，默认 2 小时（如"晚上 8-10 点"）
- 超时计算从窗口结束时算起（如窗口 8-10 点，则 10 点开始计算超时）
- **iOS 本地通知**：窗口开始时自动推送"今天还好吗？"
  - 无需服务器参与，设备本地调度，精确到分钟
  - 不依赖网络状态（离线也能触发）
- **Notification Actions（通知交互按钮）**：
  - 通知横幅下拉/长按显示"今天还好 ✓"按钮
  - 点击后直接标记回复，无需打开 App（1 次点击完成）
  - 系统自动回调 App 后台方法更新状态
- **App 内备用入口**：一键"今天还好 ✓"按钮
- 回复后温暖反馈（下次打开 App 时展示）：标题"收到，安心了" + 副标题"已收到你的平安" + "本月平安 X/Y 天"
- 下次提醒预告："明天晚上 8 点再见"
- 用户当天已回复后，不再重复推送
- **全局措辞规范**：不出现"签到""打卡"等工具化用语
- **月度统计替代连续天数**：显示"本月平安 X/Y 天"而非"连续 X 天"，避免断签焦虑

**数据/字段**：
- 提醒设置表：用户ID, 窗口开始时间, 窗口结束时间, 时区
- 回复记录表：用户ID, 回复时间, 回复方式（通知Action/App内）

**边界情况**：
- 用户未授权通知权限 → App 内显示提醒卡片 + 引导开启通知
- 设备关机/无网络 → 本地通知在设备恢复后触发（如已过窗口期则跳过，不触发误报告警）
- 用户当天已回复后再次打开 App → 显示"今天已回复"状态，不重复展示确认按钮

**验收标准**：
- [ ] 在用户设定的时间窗口内准时推送（误差 ±1 分钟，iOS 本地通知精度）
- [ ] 支持通知内快捷回复，操作 = 1 次点击（Notification Action）
- [ ] 回复后有温暖视觉反馈，无冷冰冰的系统提示
- [ ] 用户当天已回复后，不再重复推送
- [ ] 全流程不出现"签到""打卡"字样
- [ ] 显示月度统计"本月平安 X/Y 天"，不使用连续天数

### R4：超时检测与联系人通知（REQ-004）

**用户故事**：作为独居者，我希望如果我忘了回复或出了事，我的紧急联系人能收到通知来确认我的安全。

**功能行为**：
- **服务端定时器**：提醒时间窗口截止后检查数据库是否已回复
- 未回复 → 推送**关心式提醒**（APNs 远程推送） + 30 分钟宽限：
  - "没收到你的回应，有点担心，看到消息回一下？"
- 宽限后仍未回复 → 状态变为**"告警"**，通知紧急联系人：
  - **免费用户**：SMS 短信通知联系人
  - **付费用户**：SMS 短信 + 语音电话双通道
- 通知内容："[用户昵称] 今天没有回复平安，最后回复时间：XXX，请及时联系确认"
- 按联系人优先级顺序通知
- 同一告警事件短信最多 2 轮（首次 + 10 分钟后第二轮）
- 连续 3 天超时 → 提示用户调整提醒时间
- **子女端特殊逻辑**：父母不回复时标记**"未确认"**状态（非告警），通知子女"父母今天还没回复"，子女可代确认"TA没事"
- **双状态系统**："未确认"（温和提醒阶段） ≠ "告警"（联系人已通知阶段）

**通知通道策略**：

| 用户类型 | 联系人通知方式 | 单事件上限 | 单次告警成本 |
|----------|---------------|-----------|-------------|
| 免费用户 | SMS 短信 | 2 轮 SMS | ≤ 0.08 元 |
| 付费用户 | SMS + 语音电话 | 2 轮 SMS + 2 轮电话 | ≤ 0.36 元 |

**数据/字段**：
- 守护状态表：用户ID, 当前状态（正常/未确认/告警）, 最后回复时间, 告警触发时间
- 通知记录表：联系人ID, 通知时间, 通知渠道（SMS/语音电话）, 通知轮次

**边界情况**：
- 宽限期内用户回复 → 不触发联系人通知，状态恢复"正常"
- 短信发送失败 → 重试 1 次，仍失败则记录日志
- 用户连续 3 天超时 → 在 App 内弹出"提醒时间调整建议"
- 服务端宕机 → 恢复后补检查超时状态，延迟触发联系人通知（日志记录延迟）

**验收标准**：
- [ ] 提醒截止后 30 分钟宽限期内回复 → 不触发联系人通知
- [ ] 宽限期间的关心式提醒语气温暖，非系统告警风格
- [ ] 免费用户联系人收到 SMS 短信通知
- [ ] 付费用户联系人同时收到 SMS + 语音电话通知
- [ ] 宽限结束后 5 分钟内完成联系人通知发送
- [ ] 通知内容包含用户昵称、最后回复时间、建议行动
- [ ] 子女端可代确认"TA没事"，解除未确认状态
- [ ] "未确认"和"告警"状态在 UI 上有明确视觉区分
- [ ] 同一告警事件短信不超过 2 轮

### R5：联系人确认/接管（REQ-005）

**用户故事**：作为紧急联系人，我希望在确认被守护者安全后，能标记"已处理"，让系统知道情况已解决。

**功能行为**：
- 联系人通过 SMS 通知中的深链接打开 App 告警详情页
- 两个操作按钮：
  - **"已联系，TA没事 ✓"** → 告警解除，记录处理时间
  - **"联系不上，需要帮助"** → 展示建议行动清单
- 建议行动清单：拨打用户电话（一键拨号）、拨打 120（一键拨号 + 用户地址）、联系其他紧急联系人
- 所有联系人 10 分钟内均未确认 → 发送第二轮提醒（付费用户同时语音电话）
- **Phase 1 不自动拨打 120**——"需要帮助"时提供手动操作指引
- 所有操作记录日志（谁、什么时间、做了什么）
- **用户自行解除告警**：
  - 告警状态下，用户打开 App 首页显示"告警中"状态，可点击"今天还好 ✓"按钮
  - 用户回复后告警立即解除，系统通知联系人"用户已回复，TA没事"
  - 展示告警解除时间线（如：22:00 发了提醒 → 22:30 通知了联系人 → 22:42 用户回复 → 已告知联系人安全）

**联系人无 App 时的降级体验**：
- 收到 SMS 短信 → 可直接拨打用户电话确认（不需要 App）
- 如需在系统中标记"已处理" → 需下载 App 或等待后续小程序版（Phase 2）

**数据/字段**：
- 告警处理记录表：告警ID, 联系人ID, 操作（已联系/需要帮助）, 操作时间

**边界情况**：
- 多个联系人同时操作 → 第一个"已联系"即解除告警，其他人看到"告警已解除"
- 告警已解除后联系人再点击 → 显示"此告警已处理"
- 联系人未安装 App → SMS 中包含 App Store 下载链接 + "您也可以直接电话联系对方"

**验收标准**：
- [ ] 联系人可在 SMS 通知中获取告警信息并直接拨打电话
- [ ] 联系人安装 App 后可通过深链接进入确认页面
- [ ] "已联系"操作后，告警状态立即解除
- [ ] "需要帮助"提供清晰的行动建议，包括 120 拨打指引
- [ ] 10 分钟无联系人确认 → 第二轮提醒（付费用户含语音电话）
- [ ] 所有操作有记录日志
- [ ] 用户可在告警状态下自行回复"今天还好 ✓"解除告警
- [ ] 用户自行解除后，联系人收到"用户已回复"通知
- [ ] 告警解除后展示完整事件时间线

### R9：免责协议与隐私政策（REQ-009）

**用户故事**：作为用户，我需要清楚了解这个服务能做什么、不能做什么，以及我的数据如何被使用。

**功能行为**：
- 用户协议 + 隐私政策（首次使用时必读并勾选同意）
- 核心免责条款**单独高亮展示**，不藏在长篇协议中：
  - "本产品为辅助守护工具，不替代专业医疗/急救服务"
  - "本产品不保证在所有情况下都能成功通知联系人"
  - "因网络、设备、系统故障导致的延迟或失败，产品不承担责任"
- 数据使用说明：回复时间、联系人信息、短信记录、位置数据的存储和使用范围
- 数据删除入口：设置页提供"删除我的数据"功能
  - 二次确认方式：输入"确认删除"文字后才能操作，防止误触
  - 确认删除后所有数据不可恢复（回复记录、联系人、守护设置）
  - 删除提示："你的紧急联系人将不再收到通知"
- Apple 审核要求：隐私政策 URL 必须在 App Store 页面可访问

**数据/字段**：
- 协议同意记录表：用户ID, 协议版本, 同意时间

**边界情况**：
- 协议版本更新 → 下次打开时要求重新同意
- 用户拒绝更新后协议 → 服务暂停，提示"请同意最新协议以继续使用"

**验收标准**：
- [ ] 首次使用必须勾选同意后才能进入
- [ ] 免责条款单独高亮展示，不藏在长篇协议中
- [ ] 设置页提供"查看协议"和"删除我的数据"入口
- [ ] 删除数据操作需输入"确认删除"二次验证，删除后不可恢复
- [ ] 隐私政策 URL 可从 App Store 页面直接访问

### R10：紧急求助（REQ-010）

**用户故事**：作为独居者，我希望在感到不适或遇到危险时，能快速联系到紧急联系人和急救服务。

**功能行为**：
- 所有首页状态均显示"🚨 紧急求助"入口（常驻可见）
- 点击后进入紧急求助页面：
  - **一键拨打 120**（tel://120）
  - **显示当前地址**：基于 GPS 实时定位 + 用户预设的当前住址
  - **紧急联系人列表**：显示所有联系人的电话，一键拨号
  - **一键通知所有联系人**：发送 SMS "我需要帮助，地址：XXX"给所有紧急联系人
- 联系人收到求助短信后可直接拨打用户电话或 120

**数据/字段**：
- 求助记录表：用户ID, 求助时间, GPS坐标, 地址文本
- 用户表新增：当前住址（用户可选填，作为 GPS 不可用时的备用）

**边界情况**：
- GPS 定位失败 → 显示用户预设的当前住址；如未预设则提示"无法获取位置"
- 无网络 → 仍可拨打 120（电话不依赖数据网络），但无法发送 SMS 通知联系人
- 误触"紧急求助" → 联系人收到短信后，用户可再发一条"抱歉，误触了"取消

**验收标准**：
- [ ] 所有首页状态均可见"🚨 紧急求助"入口
- [ ] 一键拨打 120 功能正常
- [ ] 显示当前 GPS 地址或预设当前住址
- [ ] 一键通知所有紧急联系人（SMS）
- [ ] 求助操作有完整日志记录

### R11：暂停守护（REQ-011）

**用户故事**：作为独居者，我希望在出差、旅行或住院时暂停守护，避免产生不必要的告警。

**功能行为**：
- 设置页入口："暂停守护"
- 选择暂停时长：1 天 / 3 天 / 7 天 / 自定义（最多 30 天）
- 暂停期间：
  - 不发送每日提醒推送
  - 不触发超时告警和联系人通知
  - 首页显示"守护已暂停，X 天后恢复"+ 倒计时
  - **紧急求助按钮仍然可用**
- 到期自动恢复 → 推送"欢迎回来，守护已恢复"
- 支持提前手动恢复

**数据/字段**：
- 暂停记录表：用户ID, 暂停开始时间, 暂停结束时间, 暂停原因（可选）

**边界情况**：
- 自定义超过 30 天 → 提示"长时间暂停建议关闭守护服务"
- 暂停期间子女端看板 → 显示"TA暂停了守护"，不显示异常

**验收标准**：
- [ ] 支持 1/3/7/自定义天数的暂停
- [ ] 暂停期间不发送提醒、不触发告警
- [ ] 紧急求助功能不受暂停影响
- [ ] 到期自动恢复并推送通知
- [ ] 子女端正确显示暂停状态

```
├── ID (PK)

├── 暂停开始时间
├── 暂停结束时间
└── 暂停原因

求助记录 (HelpRequest)
```

### 8.2 iOS 系统权限

| 权限 | 用途 | 是否必须 | 降级方案 |
|------|------|---------|---------|
| 通知权限 (UNUserNotificationCenter) | 每日提醒推送、关心式提醒 | 核心（不授权=守护失效） | App 内手动确认入口 |
| 通讯录 (CNContactStore) | 导入联系人信息 | 可选 | 手动输入 |
| 位置 (CLLocationManager) | 紧急求助时提供地址 | 可选（使用时授权） | 使用预设当前住址 |
| 电话 (tel:// URL scheme) | 拨打 120、拨打联系人 | 自动（系统拨号界面） | 显示号码手动拨打 |

### 8.3 权限矩阵

| 角色 | 可查看数据 | 可操作 |
|------|-----------|--------|
| 独居者本人 | 自己的回复记录、联系人、守护状态 | 回复、管理联系人、设置提醒时间、暂停、紧急求助、删除数据 |
| 紧急联系人（免费版） | 当前守护状态（正常/告警） | 确认/接管告警 |
| 紧急联系人（付费版） | 守护状态 + 7天平安日历 | 确认/接管告警 |
| 子女 | 同上 + 代确认权限 | 代确认、管理父母设置、付费 |

### 8.4 隐私策略

- **最小数据原则**：仅采集守护功能所必需的数据
- **位置数据**：仅在用户主动触发紧急求助时获取 GPS 位置，不后台持续采集
- **数据可删除**：用户可随时删除所有数据，不可恢复
- **短信记录**：仅保留发送日志（时间、接收人），不保留短信内容
- **Apple 合规**：遵循 App Store 隐私标签要求，提供 App Privacy 详情

## 10. 验收标准汇总

### 10.1 主流程（Happy Path）

- [ ] 新用户 4 步内完成注册并进入守护首页
- [ ] 在设定时间窗口内收到"今天还好吗？"本地推送
- [ ] 通知横幅一键回复"今天还好"（Notification Action），无需打开 App
- [ ] 回复后显示温暖反馈 + 月度统计
- [ ] 超时 30 分钟宽限后，联系人收到 SMS 短信（免费）/ SMS+电话（付费）
- [ ] 联系人点击"已联系，TA没事"→ 告警立即解除
- [ ] 子女为父母开通守护 → iOS 分享邀请 → 父母一键绑定
- [ ] 紧急求助：一键 120 + 显示地址 + 通知所有联系人

### 10.2 异常路径

- [ ] 用户未授权通知权限 → App 内引导授权 + 手动确认备选
- [ ] 设备关机/离线 → 不触发误报告警
- [ ] 联系人 10 分钟未确认 → 第二轮提醒（付费用户含语音电话）
- [ ] 联系人点击"需要帮助" → 显示行动建议清单（含 120 拨打指引）
- [ ] 联系人未安装 App → SMS 通知中包含下载地址 + 可直接电话联系

### 10.3 数据一致性

- [ ] 用户回复后，守护状态 ≤ 1 分钟更新到服务端
- [ ] 告警解除后，所有联系人看到已解除状态
- [ ] 联系人变更后的下次提醒中提示用户确认
- [ ] 删除数据操作后，所有关联记录同步清除
- [ ] 暂停期间不产生任何告警或通知

### Phase 2 小程序的定位

小程序不再承担核心守护功能，而是作为**辅助入口**：
- **联系人查看看板**：联系人无需下载 App，通过小程序查看被守护者状态
