# PRD v1.1 变更记录 — 今天还好

> 变更日期：2026-06-23
> 变更原因：产品 Review 发现 16 项问题，含 4 项严重缺陷
> 影响范围：PRD v1.0 全部 9 项需求 + 原型

---

## 通知策略统一调整（影响 REQ-004 全局）

**原方案**：免费用户 → 微信模板消息通知联系人（成本 0）；付费用户 → 微信 + 短信
**新方案**：免费用户 → 短信通知联系人（每个告警事件最多 2 条，成本 0.08 元）；付费用户 → 短信 + 微信双通道
**原因**：微信模板消息要求联系人已使用过小程序或关注公众号，免费用户的联系人通常不满足条件，导致通知形同虚设
**影响**：
- REQ-004 验收标准"免费用户联系人仅收到微信通知" → 改为"免费用户联系人收到短信通知"
- 新增短信成本：0.08 元/告警事件（2条×0.04元），假设 15% 超时触发率，1000 DAU 日均成本 ≈ 12 元，可控
- 付费增值点从"有短信 vs 无短信"调整为"短信 + 微信双通道 + 更多联系人 + 关怀看板"

## 逐项修复

### FIX-01：推送内快捷回复 → 跳转小程序回复
- **问题**：微信订阅消息不支持内嵌快捷回复按钮，"≤ 1 次点击"无法达成
- **修复**：回复交互改为"点击通知 → 跳转小程序 → 点击'今天还好 ✓'"，实际 2 次点击
- **REQ-003 验收标准变更**："支持推送内快捷回复，操作 ≤ 1 次点击" → "点击通知跳转小程序后，1 次点击完成回复"
- **原型影响**：reminder 页面描述调整

### FIX-02：免费用户联系人通知改为短信
- **问题**：免费用户用微信模板消息通知联系人，但联系人大概率收不到
- **修复**：见上方"通知策略统一调整"
- **REQ-004 功能行为变更**：免费用户 = 短信（最多2条/事件）；付费用户 = 短信 + 微信模板消息
- **REQ-008 付费功能对比更新**：免费版有短信通知（基础保障），付费版增加微信通道 + 多联系人 + 看板

### FIX-03：新增"紧急地址"字段
- **问题**："需要帮助"页展示用户地址，但注册/设置流程未采集地址
- **修复**：
  - REQ-001 引导流程增加"紧急地址"步骤（选填，引导页提示"用于紧急情况下告知联系人和120"）
  - 设置页增加"紧急地址"编辑入口
  - "需要帮助"页读取用户填写的紧急地址；若未填写，显示"未设置地址，请电话告知对方您的地址"
- **数据字段**：用户表增加 `紧急地址` 字段（可选）

### FIX-04：联系人验证方式改为微信邀请
- **问题**：短信验证码冷启动，联系人不知道产品是什么，验证率低
- **修复**：联系人验证改为**微信邀请确认制**：
  1. 用户填写联系人姓名 + 手机号
  2. 系统通过小程序生成一条"守护邀请"消息，用户通过微信转发给联系人
  3. 联系人打开邀请 → 微信授权 + 确认"我愿意作为紧急联系人" → 完成绑定
  4. 联系人手机号作为备用通知渠道（短信通知不依赖微信绑定状态）
- **REQ-002 验收标准变更**："联系人手机号通过验证码校验" → "联系人通过微信邀请确认绑定；手机号作为短信通知备用渠道"
- **降级方案**：如果联系人不方便微信绑定，用户可选择"跳过绑定，仅保留手机号"（短信通知仍可用）

### FIX-05："未确认" vs "告警"互斥规则
- **问题**：同一用户既是独居者又是被子女守护的父母时，双状态触发逻辑不清
- **修复**：同一用户同一天内，**"未确认"和"告警"互斥**，按身份优先级判断：
  - 用户有守护关系（被子女守护）→ 不回复触发**"未确认"**，通知子女，子女可代确认
  - 用户无守护关系（普通独居者）→ 不回复触发**"告警"**，通知紧急联系人
  - 用户同时有守护关系和紧急联系人 → 优先触发**"未确认"**通知子女；子女未代确认且超时 → 再触发**"告警"**通知紧急联系人（两级递进）
- **REQ-004 功能行为补充**：新增双状态互斥规则和两级递进逻辑

### FIX-06：子女代确认不等于用户回复
- **问题**：子女代确认"TA没事"后，超时检测引擎是否视为已回复？
- **修复**：
  - 子女代确认 → 解除"未确认"状态，**但不计入用户的日回复记录**
  - 关怀看板分两行展示："自己回复 X 天" + "子女代确认 Y 天"
  - 连续 3 天用户本人未回复（即使子女代确认了）→ 提示子女"建议调整提醒时间或联系父母确认"
- **REQ-004 + REQ-007 功能行为补充**

### FIX-07：子女权限边界定义
- **问题**：子女可管理父母设置，但权限边界未定义
- **修复**：新增子女权限矩阵：

| 操作 | 子女可做 | 仅父母本人可做 |
|------|---------|---------------|
| 修改提醒时间窗口 | ✓ | - |
| 添加/修改紧急联系人 | ✓ | - |
| 代确认"TA没事" | ✓ | - |
| 为父母付费订阅 | ✓ | - |
| 修改父母昵称/手机号 | ✗ | ✓ |
| 删除父母守护账户 | ✗ | ✓ |
| 修改父母紧急地址 | ✗ | ✓ |
| 取消父母订阅 | ✗ | ✓ |
| 暂停/恢复守护 | ✗ | ✓ |
| 删除父母数据 | ✗ | ✓ |

- **REQ-006 功能行为补充**：新增权限边界说明

### FIX-08：首页状态优先级逻辑
- **问题**：5 种首页状态缺少判定优先级
- **修复**：打开小程序时按以下优先级判定显示哪个状态：

| 优先级 | 状态 | 判定条件 |
|--------|------|---------|
| 1（最高） | 告警中（home-alert） | 当前有未解除的告警 |
| 2 | 宽限期（home-grace） | 已过提醒窗口截止 + 30分钟宽限内 + 未回复 |
| 3 | 已回复（home-done） | 今天已回复 |
| 4 | 等待回复（home） | 当前在提醒时间窗口内 + 未回复 |
| 5（最低） | 非时段（home-idle） | 当前不在提醒窗口内 + 无告警 + 未回复 |

- **额外规则**：守护暂停时，所有状态替换为"守护已暂停"页面
- **REQ-003 功能行为补充**：新增状态判定逻辑

### FIX-09：连续天数 → 月度平安统计
- **问题**："连续 X 天平安"断签引发焦虑，与安全守护的温暖调性矛盾
- **修复**：
  - "连续 X 天平安" → **"本月平安 X/Y 天"**（月度统计，不会因断签清零）
  - 保留 streak-badge 组件但改为月度展示："本月平安 28/30 天"
  - 关怀看板同步调整为月度视图
- **REQ-003 功能行为变更**：回复反馈文案从"已连续 X 天平安" → "本月平安 X/Y 天"
- **原型影响**：所有 streak-badge 改为月度统计

### FIX-10：移除紧急场景下的付费引导
- **问题**："需要帮助"页面中"联系其他联系人"项显示"免费版仅支持1位 → 升级"
- **修复**：
  - "需要帮助"页面移除所有付费引导
  - 免费用户也展示紧急联系人列表（即使只有 1 位），提供一键拨号
  - 付费引导仅在设置页、关怀看板等**非紧急场景**出现
- **原型影响**：alert-help 页面重构

### FIX-11：新增主动"紧急求助"入口
- **问题**：用户清醒但受伤/生病时，没有主动求助入口
- **修复**：
  - 首页底部增加常驻小按钮 **"紧急求助"**（红色，不显眼但可发现）
  - 点击后进入紧急求助页面：
    - 一键拨打 120（显示紧急地址）
    - 一键通知所有紧急联系人（发送"我需要帮助"短信）
    - 不需要等待超时触发，用户主动发起
  - 触发后联系人收到特殊通知："[用户] 主动发起紧急求助，请立即联系！"
- **新增 REQ-010：紧急求助（P0）**
- **原型影响**：所有首页状态增加紧急求助按钮 + 新增 emergency 页面

### FIX-12：新增"暂停守护"功能
- **问题**：用户出差/住院/与朋友同住时，每日提醒 + 告警是纯打扰
- **修复**：
  - 设置页增加"暂停守护"入口
  - 可选暂停时长：1天 / 3天 / 7天 / 自定义日期
  - 暂停期间：不发送提醒、不触发超时、不通知联系人
  - 首页显示"守护已暂停，X天后自动恢复"+ 一键"立即恢复"
  - 暂停到期自动恢复，发送"守护已恢复，今晚 X 点见"通知
  - 暂停记录保留在关怀看板中（标记为"暂停"而非"未回复"）
- **新增 REQ-011：暂停守护（P1）**
- **原型影响**：新增 pause 页面 + home-paused 状态

### FIX-13：引导流程 7 步 → 5 步
- **问题**：7 步引导太长，协议页 3 条免责条款在用户未体验价值前造成劝退
- **修复**：
  - 原流程：登录 → 协议 → 基本信息 → 联系人 → 提醒时间 → 订阅授权 → 完成（7步）
  - 新流程：**登录+协议 → 联系人 → 提醒时间+地址 → 完成（含授权引导）→ 首页**（4步核心 + 1步可选）
  - 具体合并：
    - 登录页底部保留"登录即表示同意协议"链接（不再单独成页）
    - 基本信息（昵称/手机号）从微信 profile 自动获取，无需手动填写
    - 订阅授权移到完成页，作为"开启提醒"按钮的一部分（点击授权 = 开启）
    - 紧急地址合并到提醒时间页（选填）
  - 步骤指示器从 5 点改为 3 点：联系人 → 提醒设置 → 完成
- **REQ-001 验收标准变更**："微信授权后 3 步内完成注册流程" → "微信授权后 3 步内完成引导（联系人 → 提醒设置 → 完成）"

### FIX-14：日历星期对齐
- **问题**：2026年6月22日是周一，原型日历排列错误
- **修复**：日历以周一为起始，6月22日（周一）正确定位
- **原型影响**：child-dashboard 日历重排

### FIX-15：告警时间线增加"关心式提醒"节点
- **问题**：联系人视角的时间线缺少 22:00-22:30 之间的关心式提醒环节
- **修复**：联系人告警页时间线增加一个节点：
  - 22:00 给小李发了"今天还好吗？"
  - **22:00 小李没回复，发了关心式提醒**（新增）
  - 22:30 小李仍未回复，通知了你
  - 正在等你确认...
- **原型影响**：alert-contact 时间线增加一行

### FIX-16：添加联系人后增加保存确认
- **问题**：contact-form 点保存后直接跳回列表，无操作确认
- **修复**：contact-form 保存后跳转回列表页，顶部显示 toast "联系人已添加，邀请已发送"
- **原型影响**：onboarding-4 页面增加 toast 提示

## 付费功能对比更新（REQ-008）

| 功能 | 免费版 | 守护版（9.9元/月） |
|------|--------|-------------------|
| 每日"今天还好"提醒 | ✓ | ✓ |
| 紧急联系人数量 | 1 位 | 最多 5 位 |
| 联系人通知 | 短信（2条/事件） | 短信 + 微信双通道 |
| 联系人优先级排序 | ✗ | ✓ |
| 关怀看板 | 仅当前状态 | 7天平安日历 |
| 子女代付费 | - | ✓ |
| 紧急求助 | ✓ | ✓ |
| 暂停守护 | ✓ | ✓ |

**关键变化**：免费版也有短信通知联系人（基础安全保障），付费版增加微信通道和更多高级功能。确保免费用户的核心守护链路可用。

## 数据模型补充

```
用户 (User) — 新增字段
├── 紧急地址（可选）
└── 守护暂停到期时间（可选）

联系人 (Contact) — 修改字段
├── 绑定方式：微信邀请确认 / 仅手机号
└── 绑定状态：已绑定 / 待确认 / 仅手机号

告警处理记录 (AlertAction) — 新增字段
├── 告警来源：超时触发 / 用户主动求助
└── 代确认标记：是否由子女代确认

回复记录 (ReplyLog) — 新增字段
└── 回复类型：本人回复 / 子女代确认
```
