## 一、全局信息架构

### 1.1 TabBar 导航（新增）

原型的页面跳转全部依赖文字链接或按钮，缺少全局导航骨架。建议增加底部 TabBar：

```
TabBar
├── 首页（守护核心入口）
├── 关怀看板（联系人/子女视角，免费用户可见但需升级解锁详情）
└── 设置（联系人管理、提醒设置、订阅、协议、数据删除）
```

**设计意图**：
- 首页承载每日核心操作（一键回复）
- 关怀看板从首页的隐藏链接提升为一级入口，增加曝光
- 设置收口所有配置项，降低首页认知负荷

## 二、引导流程组件树（Onboarding）

### 2.1 步骤指示器修正

```
StepIndicator
├── 显示范围：仅在"协议→基本信息→联系人→提醒时间→授权"5步中展示
├── 登录页、完成页：不显示步骤指示器（这两步是入口/出口，不属于引导进度）
├── Props
│   ├── currentStep: 1-5
│   └── totalSteps: 5
└── 视觉
    ├── done: 实心绿色圆点
    ├── active: 绿色胶囊（宽 24px）
    └── pending: 灰色圆点
```

**修正说明**：原型的 7 步引导中，"登录"是系统行为，"完成"是结果页，用户实际需要感知的进度是中间 5 步。步骤指示器对齐到用户可操作的 5 步，减少认知负担。

## 八、通用基础组件库

以下是跨页面复用的基础组件清单：

### 8.1 布局类

| 组件 | 用途 | 使用页面 |
|------|------|----------|
| StatusBar | 品牌顶栏（绿色/白色/红色变体） | 全部页面 |
| BackHeader | 带返回按钮的子页顶栏 | 子页面 |
| PageHeader | 页面标题 + 副标题 | 引导流程、设置子页 |
| Card | 白色圆角卡片容器 | 全部页面 |
| Divider | 分割线 | 卡片内分隔 |

### 8.2 表单类

| 组件 | 用途 | 使用页面 |
|------|------|----------|
| FormField | label + input + 校验提示 | 基本信息、联系人表单 |
| TimeWindowPicker | 时间窗口选择（开始-结束） | 提醒时间、子女代设 |
| AgreementCheckbox | 带文本的勾选框 | 协议页 |

### 8.3 反馈类

| 组件 | 用途 | 使用页面 |
|------|------|----------|
| AlertBanner | 信息/警告/危险横幅（ok/warn/danger） | 引导流程、告警页、首页 |
| Tag | 状态标签（已验证/正常/告警中） | 联系人卡片、看板 |
| Toast | 轻量操作反馈 | 回复成功、保存成功 |
| Dialog | 二次确认对话框 | 授权拒绝、撤回回复、删除数据 |
| Skeleton | 加载骨架屏 | 首页、看板、设置 |

### 8.4 操作类

| 组件 | 用途 | 使用页面 |
|------|------|----------|
| Button（primary/outline/ghost/danger/warm） | 按钮变体 | 全部页面 |
| ConfirmButton（大圆按钮） | 核心操作入口 | 首页 |
| ActionItem | 带图标+描述的操作行 | 需要帮助页、设置页 |
| ContactCard | 联系人信息展示卡片 | 联系人列表、告警页 |

### 8.5 数据展示类

| 组件 | 用途 | 使用页面 |
|------|------|----------|
| StreakBadge | 连续平安天数徽章 | 首页、已回复页 |
| CalendarGrid | 7 列平安日历 | 关怀看板 |
| Timeline | 垂直时间线 | 告警通知、确认安全 |
| StatsRow | label-value 统计行 | 看板、设置 |
| EmptyState | 空态插画 + 文案 + 引导 | 联系人空列表、看板空数据 |
| ErrorState | 错误态插画 + 重试按钮 | 首页、看板、设置 |

## 十、与原型差异汇总

| # | 审查问题 | 本方案修正 | 涉及组件 |
|---|---------|-----------|---------|
| 1 | 步骤指示器 5 dot vs 7 步 | 登录页/完成页去掉指示器，5 步对应 5 dot | StepIndicator |
| 2 | 协议默认全选 | 默认未勾选，按钮联动禁用态 | AgreementCheckbox + SubmitButton |
| 3 | 联系人无验证流程 | 新增 ContactFormModal 含验证码 | ContactFormModal |
| 4 | 授权拒绝无差异 | 新增 AuthDeniedConfirmDialog + 完成页状态区分 | AuthDeniedConfirmDialog, Complete |
| 5 | 首页仅单状态 | 5 种状态分支（等待/已回复/宽限/告警/非时段） | HomeScreen |
| 6 | 大圆按钮缺 active 态 | 新增 active/loading/disabled 态 | ConfirmButton |
| 7 | 已回复无撤回/帮助出口 | 新增 UndoSection + NotFeelingWellSection | ConfirmedScreen |
| 8 | 告警缺时间线/倒计时 | 新增 AlertTimeline + NextRoundCountdown | AlertContactScreen |
| 9 | "暂无其他联系人" | 改为付费引导 + UpgradeLink | AlertHelpScreen |
| 10 | 用户侧缺告警解除通知 | 首页状态 D 覆盖 + ConfirmedScreen 扩展 | HomeScreen(状态D) |
| 11 | 子女端流程不完整 | 拆分为 3 步（填写/发送/等待绑定） | ChildSetupScreen |
| 12 | 日历星期对齐错误 | CalendarGrid 首行补 EmptyCell | DashboardScreen |
| 13 | 缺 loading/empty/error | 新增 Skeleton、EmptyState、ErrorState 通用组件 | 全局 |
| 14 | 缺全局导航 | 新增底部 TabBar（首页/看板/设置） | TabBar |
| 15 | 告警事件缺日期 | 新增 EventDateLabel | AlertConfirmScreen |
