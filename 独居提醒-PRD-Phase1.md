> 版本：v1.0
> 创建日期：2026-06-22
> 状态：待评审
> 产品经理：小产

---

## 1. 概述

**今天还好**是一款面向独居人士的微信小程序，通过每日温暖的平安确认 + 超时自动通知紧急联系人，构建轻量化的独居安全守护网络。Phase 1 MVP 以最小成本验证"每日确认→超时通知→联系人接管"核心链路的真实需求和用户留存。

**核心价值主张**：不叫"签到"，不叫"打卡"——每天问一句"今天还好吗？"，让独居者用一次点击告诉关心自己的人"我没事"。

## 2. 背景与证据

### 2.1 市场背景

- 中国独居人口约 **1.2 亿**，且持续增长
- 独居者面临的核心焦虑："如果我出事了，多久才会有人知道？"
- 独居老人/慢性病患者的安全问题是家庭和社会痛点

### 2.2 竞品空白

| 竞品痛点 | 证据来源 |
|----------|----------|
| 通知渠道单一（仅邮件） | 死了么 App 评测 |
| 签到周期不合理（3天才触发） | 死了么 App 功能分析 |
| 无穿戴设备联动 | 全品类竞品扫描 |
| 缺乏主动预警 | 竞品功能对比 |
| 体验粗糙，缺乏信任感设计 | 独居签到/活着么 UI 分析 |

### 2.3 产品机会

现有竞品停留在"被动签到工具"层面，我们的差异化在于：
- **温暖交互**：以"今天还好"替代"签到"，降低心理负担
- **分级通知**：免费/付费分通道，短信成本可控
- **子女生态**：子女付费为父母开通，情感驱动付费
- **Phase 2 演进**：Apple Watch 心率联动 + 分级响应链（自动拨打120）

## 4. 用户与场景

### 4.1 目标用户

| 用户角色 | 描述 | 核心需求 |
|----------|------|----------|
| **独居者本人** | 独居青年/中年/老人 | 出事时有人知道；心理安全感 |
| **紧急联系人** | 家人、朋友、邻居 | 被守护者出事时能被及时通知 |
| **子女** | 远在他城的子女 | 远程守护独居父母；主动查看安全状态 |

### 4.2 核心场景

**场景 A：独居者日常守护**
> 小李，28岁，独居。每天晚上 8 点收到微信消息"今天还好吗？"，点一下"今天还好 ✓"，看到"收到，安心了"的反馈。整个过程不到 5 秒。如果某天忘了回复，系统先温和提醒，超时后自动通知她妈妈。

**场景 B：超时告警与联系人接管**
> 小李今天没回复。10:30（窗口截止+30分钟宽限后），系统通知妈妈："小李今天没有回复平安，最后回复时间：昨天 20:15，请及时联系确认。"妈妈打电话确认小李安全后，在小程序点"已联系，TA没事 ✓"，告警解除。

**场景 C：子女远程守护**
> 小张在深圳工作，妈妈在老家独居。小张在小程序"为家人开通守护"，设置妈妈每天 7-9 点提醒。妈妈不常回复，但不触发告警——系统标记"未确认"，小张收到"妈妈今天还没回复"，小张打电话确认后在小程序代确认"TA没事"。小张付费解锁关怀看板，随时看妈妈的平安日历。

**场景 D：需要帮助**
> 联系人收到告警通知后联系不上被守护者，在小程序点"联系不上，需要帮助"，系统显示建议行动清单：拨打用户电话、拨打 120（提供用户地址）、联系其他紧急联系人。Phase 1 不自动拨打 120，由联系人手动操作。

### 5.3 假设

1. 微信小程序订阅消息能满足每日提醒的频次需求（需验证微信平台策略）
2. 微信模板消息能有效触达紧急联系人
3. 用户愿意为"多联系人 + 短信通知 + 关怀看板"付费 9.9 元/月
4. 子女付费意愿强于独居者本人付费
5. 云端定时器（如微信云开发定时触发器）能满足超时检测的精度和可靠性

### 5.4 依赖

| 依赖项 | 风险 | 缓解 |
|--------|------|------|
| 微信订阅消息授权率 | 用户可能拒绝授权，导致无法推送 | 引导流程强调不授权=无守护；小程序内备用入口 |
| 微信模板消息送达率 | 联系人未关注可能收不到 | 付费用户增加短信兜底 |
| 短信服务商稳定性 | 短信发送失败 | 选择主流服务商（如阿里云短信）+ 重试机制 |
| 微信支付审核 | 订阅付费需通过微信审核 | 提前准备资质，Phase 1 初期可先用一次性付费过渡 |

## 6. 用户流程

### 6.1 首次使用流程（独居者本人）

```
打开小程序
  │
  ▼
微信一键授权登录
  │
  ▼
阅读并同意免责协议 + 隐私政策（必须勾选）
  │
  ▼
填写基本信息（昵称、手机号）
  │
  ▼
设置紧急联系人（至少1人）
  ├─ 输入联系人姓名、手机号、关系
  ├─ 发送短信验证码
  └─ 验证通过 → 保存
  │
  ▼
设置提醒时间窗口（默认：晚上 8-10 点）
  │
  ▼
引导授权订阅消息
  ├─ 授权 → "太好了！每天晚上会收到'今天还好吗？'"
  └─ 拒绝 → "不授权将无法收到每日平安提醒，建议开启"
  │
  ▼
进入守护首页
  └─ 显示："守护已开启，今晚 8 点见"
```

### 6.2 每日提醒流程

```
到达提醒时间窗口起始时间
  │
  ▼
推送微信订阅消息："今天还好吗？"
  │
  ├─ 用户在推送内快捷回复 → 状态更新为"已回复"
  │   └─ 显示温暖反馈："收到，安心了 ☺ 已连续 X 天平安"
  │   └─ 预告："明天晚上 8 点再见"
  │
  ├─ 用户打开小程序 → 点击"今天还好 ✓" → 同上
  │
  └─ 用户未回复
      │
      ▼
  时间窗口结束 → 进入 30 分钟宽限
      │
      ▼
  推送关心式提醒："没收到你的回应，有点担心，看到消息回一下？"
      │
      ├─ 用户回复 → 状态正常，不触发联系人通知
      │
      └─ 仍未回复 → 30 分钟宽限结束
          │
          ▼
      状态变为"告警" → 通知紧急联系人
          ├─ 免费用户：微信模板消息
          └─ 付费用户：微信 + 短信
```

### 6.3 联系人通知与接管流程

```
联系人收到通知
  │
  ▼
点击通知 → 进入小程序告警详情页
  │
  ├─ 点击"已联系，TA没事 ✓"
  │   └─ 告警解除，记录处理时间
  │   └─ 通知用户："你的联系人已确认你安全"
  │
  └─ 点击"联系不上，需要帮助"
      └─ 显示建议行动清单：
          ├─ 拨打用户电话 [一键拨号]
          ├─ 拨打 120 [一键拨号 + 显示用户地址]
          └─ 联系其他紧急联系人 [列表]
      │
  10 分钟内所有联系人均未确认
      └─ 发送第二轮短信提醒给联系人（仅付费用户）
```

```
子女打开小程序 → 选择"为家人开通守护"

  ▼

  │
  ▼
生成邀请链接/二维码 → 发送给父母
  │
  ▼
父母点击链接 → 微信授权绑定 → 一键完成

  ▼

  │
  ▼

  ├─ 连续未回复天数

```

## 7. 功能需求

### R1：用户注册与身份绑定（REQ-001）

**用户故事**：作为独居者，我希望用微信一键登录并填写基本信息，以便快速开始使用守护服务。

**功能行为**：
- 微信一键登录，获取 OpenID + 头像 + 昵称
- 补充填写：昵称（可修改微信默认）、手机号（用于短信通知联系人时识别身份）
- 首次引导流程：登录 → 协议 → 基本信息 → 联系人 → 提醒时间 → 订阅授权 → 完成
- 未完成引导的用户下次打开时从断点继续

**数据/字段**：
- 用户表：openID, 昵称, 手机号, 注册时间, 引导完成状态, 订阅授权状态, 付费状态

**边界情况**：
- 用户中途退出引导 → 下次打开从断点继续
- 手机号已被其他账号使用 → 提示"该手机号已绑定其他账号"

**验收标准**：
- [ ] 微信授权后 3 步内完成注册流程
- [ ] 未授权订阅消息时，明确提示"不授权将无法收到每日平安提醒"
- [ ] 注册完成后自动进入守护首页
- [ ] 中途退出后再次打开可从断点继续

---

### R2：紧急联系人管理（REQ-002）

**用户故事**：作为独居者，我希望设置至少一个紧急联系人，确保出事时有人被通知。

**功能行为**：
- 免费用户：最多 1 个紧急联系人
- 付费用户：最多 5 个紧急联系人 + 优先级排序
- 联系人信息字段：姓名、手机号、与用户关系、微信 OpenID（可选，用于微信通知）
- 手机号通过短信验证码校验有效性
- 联系人变更（删除/修改）后，下次提醒时提示用户确认联系人信息是否仍然正确
- 联系人也可通过小程序链接注册/关联微信身份

**数据/字段**：
- 联系人表：ID, 用户ID, 姓名, 手机号, 关系, 微信OpenID, 优先级, 验证码状态, 创建时间

**边界情况**：
- 免费用户尝试添加第 2 个联系人 → 展示付费引导（不阻断使用，提示"升级可添加更多"）
- 联系人手机号验证失败 → 提示重新输入
- 用户删除所有联系人 → 阻断守护服务，要求至少保留 1 个

**验收标准**：
- [ ] 至少设置 1 个紧急联系人后才能开始使用守护服务
- [ ] 联系人手机号通过验证码校验
- [ ] 免费用户添加第 2 个联系人时展示付费引导，不阻断核心功能
- [ ] 联系人被删除/变更后，下次提醒时提示用户确认联系人信息

---

### R3：每日"今天还好"提醒（REQ-003）

**用户故事**：作为独居者，我希望每天收到一条像朋友问候一样的提醒，点一下就能告诉关心我的人我没事。

**功能行为**：
- 用户设定每日提醒时间窗口，默认 2 小时（如"晚上 8-10 点"）
- 超时计算从窗口结束时算起（如窗口 8-10 点，则 10 点开始计算超时）
- 窗口开始时推送微信订阅消息："今天还好吗？"
- **推送内快捷回复**：用户在微信通知中直接回复"今天还好"，无需打开小程序
- 小程序内备用入口：一键"今天还好 ✓"按钮
- 回复后温暖反馈："收到，安心了 ☺" + "已连续 X 天平安"
- 下次提醒预告："明天晚上 8 点再见"
- 用户当天已回复后，不再重复推送
- **全局措辞规范**：不出现"签到""打卡"等工具化用语

**数据/字段**：
- 提醒设置表：用户ID, 窗口开始时间, 窗口结束时间, 时区
- 回复记录表：用户ID, 回复时间, 回复方式（推送/小程序）

**边界情况**：
- 用户未授权订阅消息 → 小程序内显示提醒卡片，引导授权
- 推送发送失败（微信平台故障） → 记录日志，不触发超时告警
- 用户当天已回复后再次打开小程序 → 显示"今天已回复"状态，不重复展示确认按钮

**验收标准**：
- [ ] 在用户设定的时间窗口内准时推送（误差 ±15 分钟）
- [ ] 支持推送内快捷回复，操作 ≤ 1 次点击
- [ ] 回复后有温暖视觉反馈，无冷冰冰的系统提示
- [ ] 用户当天已回复后，不再重复推送
- [ ] 全流程不出现"签到""打卡"字样

---

### R4：超时检测与联系人通知（REQ-004）

**用户故事**：作为独居者，我希望如果我忘了回复或出了事，我的紧急联系人能收到通知来确认我的安全。

**功能行为**：
- 云端定时器：提醒时间窗口截止后检查是否已回复
- 未回复 → 推送**关心式提醒** + 30 分钟宽限："没收到你的回应，有点担心，看到消息回一下？"
- 宽限后仍未回复 → 状态变为**"告警"**，通知紧急联系人：
  - **免费用户**：仅微信模板消息（成本为 0）
  - **付费用户**：微信模板消息 + 短信双通道
- 通知内容："[用户昵称] 今天没有回复平安，最后回复时间：XXX，请及时联系确认"
- 按联系人优先级顺序通知
- 同一告警事件短信最多 2 轮（首次 + 10 分钟后第二轮）
- 连续 3 天超时 → 提示用户调整提醒时间
- **子女端特殊逻辑**：父母不回复时标记**"未确认"**状态（非告警），通知子女"父母今天还没回复"，子女可代确认"TA没事"
- **双状态系统**："未确认"（温和提醒阶段） ≠ "告警"（联系人已通知阶段）

**数据/字段**：
- 守护状态表：用户ID, 当前状态（正常/未确认/告警）, 最后回复时间, 告警触发时间
- 通知记录表：联系人ID, 通知时间, 通知渠道（微信/短信）, 通知轮次

**边界情况**：
- 宽限期内用户回复 → 不触发联系人通知，状态恢复"正常"
- 免费用户联系人未关注公众号 → 仅记录"微信通知未送达"，无短信兜底（付费引导点）
- 短信发送失败 → 重试 1 次，仍失败则记录日志
- 用户连续 3 天超时 → 在小程序内弹出"提醒时间调整建议"

**验收标准**：
- [ ] 提醒截止后 30 分钟宽限期内回复 → 不触发联系人通知
- [ ] 宽限期间的关心式提醒语气温暖，非系统告警风格
- [ ] 免费用户联系人仅收到微信通知，不产生短信成本
- [ ] 付费用户联系人同时收到微信 + 短信通知
- [ ] 宽限结束后 5 分钟内完成联系人通知发送
- [ ] 通知内容包含用户昵称、最后回复时间、建议行动
- [ ] 联系人收到通知后可一键进入小程序查看状态
- [ ] 子女端可代确认"TA没事"，解除未确认状态
- [ ] "未确认"和"告警"状态在 UI 上有明确视觉区分
- [ ] 同一告警事件短信不超过 2 轮

### R5：联系人确认/接管（REQ-005）

**用户故事**：作为紧急联系人，我希望在确认被守护者安全后，能标记"已处理"，让系统知道情况已解决。

**功能行为**：
- 联系人通过通知消息中的链接进入小程序告警详情页
- 两个操作按钮：
  - **"已联系，TA没事 ✓"** → 告警解除，记录处理时间
  - **"联系不上，需要帮助"** → 展示建议行动清单
- 建议行动清单：拨打用户电话（一键拨号）、拨打 120（一键拨号 + 用户地址）、联系其他紧急联系人
- 所有联系人 10 分钟内均未确认 → 发送第二轮短信提醒（仅付费用户）
- **Phase 1 不自动拨打 120**——"需要帮助"时提供手动操作指引
- 所有操作记录日志（谁、什么时间、做了什么）

**数据/字段**：
- 告警处理记录表：告警ID, 联系人ID, 操作（已联系/需要帮助）, 操作时间

**边界情况**：
- 多个联系人同时操作 → 第一个"已联系"即解除告警，其他人看到"告警已解除"
- 告警已解除后联系人再点击 → 显示"此告警已处理"
- 联系人未注册小程序 → 引导简单注册（仅需微信授权 + 手机号）

**验收标准**：
- [ ] 联系人可在通知消息中直接进入确认页面
- [ ] "已联系"操作后，告警状态立即解除
- [ ] "需要帮助"提供清晰的行动建议，包括 120 拨打指引
- [ ] 10 分钟无联系人确认 → 第二轮短信提醒（付费用户）
- [ ] 所有操作有记录日志（谁在什么时间做了什么）

### R6：子女端 — 为父母开通守护（REQ-006）

**用户故事**：作为子女，我希望能为远方的父母设置守护服务，随时了解他们的安全状态。

**功能行为**：
- 子女登录后选择"为家人开通守护"入口
- 填写父母信息（姓名、手机号）
- 生成邀请链接/二维码 → 发送给父母微信
- 父母点击链接 → 微信授权绑定 → 完成
- 子女自动成为父母的第一紧急联系人
- 子女可代为设置提醒时间、管理联系人
- 父母不回复时：标记"未确认"状态，通知子女，子女可代确认"TA没事"

**数据/字段**：
- 守护关系表：子女用户ID, 父母用户ID, 关系类型, 创建时间

**边界情况**：
- 父母已自行注册 → 邀请链接提示"该用户已注册，是否关联守护？"
- 父母未绑定邀请链接 → 子女端显示"等待父母绑定"

**验收标准**：
- [ ] 子女端创建守护档案流程 ≤ 5 步
- [ ] 父母通过邀请链接一键绑定，无需重复填写信息
- [ ] 子女可远程管理父母的守护设置
- [ ] 父母侧有明确提示"XXX 正在守护你"

### R7：关怀看板（REQ-007）

**用户故事**：作为子女，我希望随时能看到父母最近的平安回复记录和安全状态，不用等到超时告警才知道。

**功能行为**（付费用户解锁）：
- 最近 7 天平安日历（哪天回复了、哪天没回复）
- 连续未回复天数高亮提醒
- 最后回复时间
- 当前守护状态（正常/未确认/告警中）
- 免费用户联系人仅能看到"当前状态正常/告警中"

**数据/字段**：
- 复用回复记录表，查询维度为联系人查看的被守护者数据

**边界情况**：
- 付费到期 → 看板降级为免费版视图，提示续费
- 被守护者删除联系人 → 联系人看板同步失效，提示"您已不再是对方的紧急联系人"

**验收标准**：
- [ ] 免费用户联系人只能看到"当前状态正常/告警中"
- [ ] 付费用户联系人可查看详细平安回复历史
- [ ] 数据实时更新，回复后 ≤ 1 分钟同步到看板
- [ ] 看板入口在联系人小程序首页可见

### R8：订阅付费（REQ-008）

**用户故事**：作为用户/子女，我愿意为更完善的守护服务付费，获得更多联系人和更及时的通知。

**功能行为**：
- 免费版 vs 付费版功能对比页（清晰展示差异）
- 微信支付连续包月订阅：9.9 元/月起
- 付费解锁项：多联系人（≤5）、关怀看板、短信通知通道
- 子女可为父母订阅付费（代付模式）
- 新用户 7 天免费试用，到期前 1 天提醒
- 可随时取消订阅，取消后保留当月权益至到期

**数据/字段**：
- 订阅表：用户ID, 订阅状态, 开始时间, 到期时间, 自动续费, 付费者（本人/子女）

**边界情况**：
- 微信支付失败 → 提示重试，不立即降级
- 试用期到期未付费 → 自动降级为免费版，已添加的多余联系人保留但不可编辑

**验收标准**：
- [ ] 付费流程 ≤ 3 步完成
- [ ] 明确展示免费/付费功能差异
- [ ] 支持微信支付连续包月
- [ ] 提供 7 天免费试用，到期前提醒
- [ ] 可随时取消订阅，取消后保留当月权益

### R9：免责协议与隐私政策（REQ-009）

**用户故事**：作为用户，我需要清楚了解这个服务能做什么、不能做什么，以及我的数据如何被使用。

**功能行为**：
- 用户协议 + 隐私政策（首次使用时必读并勾选同意）
- 核心免责条款**单独高亮展示**，不藏在长篇协议中：
  - "本产品为辅助守护工具，不替代专业医疗/急救服务"
  - "本产品不保证在所有情况下都能成功通知联系人"
  - "因网络、设备、系统故障导致的延迟或失败，产品不承担责任"
- 数据使用说明：回复时间、联系人信息、短信记录的存储和使用范围
- 数据删除入口：设置页提供"删除我的数据"功能，二次确认后不可恢复

**数据/字段**：
- 协议同意记录表：用户ID, 协议版本, 同意时间

**边界情况**：
- 协议版本更新 → 下次打开时要求重新同意
- 用户拒绝更新后协议 → 服务暂停，提示"请同意最新协议以继续使用"

**验收标准**：
- [ ] 首次使用必须勾选同意后才能进入
- [ ] 免责条款单独高亮展示，不藏在长篇协议中
- [ ] 设置页提供"查看协议"和"删除我的数据"入口
- [ ] 删除数据操作二次确认，删除后不可恢复

## 8. 数据与权限要求

### 8.1 数据模型概览

```
用户 (User)
├── openID (PK)
├── 昵称
├── 手机号
├── 提醒窗口（开始/结束）
├── 订阅状态
├── 订阅授权状态
└── 引导完成状态

联系人 (Contact)
├── ID (PK)
├── 用户ID (FK)
├── 姓名、手机号、关系
├── 微信OpenID
├── 优先级
└── 验证码状态

回复记录 (ReplyLog)
├── ID (PK)
├── 用户ID (FK)
├── 回复时间
└── 回复方式

守护状态 (GuardStatus)
├── 用户ID (PK)
├── 当前状态（正常/未确认/告警）
├── 最后回复时间
└── 告警触发时间

通知记录 (NotificationLog)
├── ID (PK)
├── 联系人ID (FK)
├── 通知时间、渠道、轮次
└── 送达状态

守护关系 (GuardRelation)
├── 子女用户ID
├── 父母用户ID
└── 关系类型

告警处理记录 (AlertAction)
├── 告警ID (FK)
├── 联系人ID (FK)
├── 操作（已联系/需要帮助）
└── 操作时间
```

### 8.2 权限矩阵

| 角色 | 可查看数据 | 可操作 |
|------|-----------|--------|
| 独居者本人 | 自己的回复记录、联系人、守护状态 | 回复、管理联系人、设置提醒时间、删除数据 |
| 紧急联系人（免费版） | 当前守护状态（正常/告警） | 确认/接管告警 |
| 紧急联系人（付费版） | 守护状态 + 7天平安日历 | 确认/接管告警 |
| 子女 | 同上 + 代确认权限 | 代确认、管理父母设置、付费 |

### 8.3 隐私策略

- **最小数据原则**：仅采集守护功能所必需的数据
- **本地优先**：健康数据（Phase 2）仅在设备本地处理
- **位置数据**：Phase 1 不主动采集位置；仅在告警触发时获取用户最后打开小程序时的位置
- **数据可删除**：用户可随时删除所有数据，不可恢复
- **短信记录**：仅保留发送日志（时间、接收人），不保留短信内容

## 9. 验收标准汇总

### 9.1 主流程（Happy Path）

- [ ] 新用户 3 步内完成注册并进入守护首页
- [ ] 在设定时间窗口内收到"今天还好吗？"推送（误差 ±15 分钟）
- [ ] 推送内一键回复"今天还好"，收到温暖反馈
- [ ] 超时 30 分钟宽限后，联系人收到通知（免费=微信/付费=微信+短信）
- [ ] 联系人点击"已联系，TA没事"→ 告警立即解除
- [ ] 子女为父母开通守护 → 父母一键绑定 → 子女可查看状态

### 9.2 异常路径

- [ ] 用户未授权订阅消息 → 小程序内引导授权
- [ ] 推送发送失败 → 记录日志，不触发误报告警
- [ ] 联系人 10 分钟未确认 → 第二轮提醒（付费用户短信）
- [ ] 联系人点击"需要帮助" → 显示行动建议清单（含 120 拨打指引）
- [ ] 免费用户联系人未关注公众号 → 记录未送达，展示付费引导

### 9.3 数据一致性

- [ ] 用户回复后，守护状态 ≤ 1 分钟更新
- [ ] 告警解除后，所有联系人看到已解除状态
- [ ] 联系人变更后的下次提醒中提示用户确认
- [ ] 删除数据操作后，所有关联记录同步清除
