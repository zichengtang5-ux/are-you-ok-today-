# PRD v2.0 变更记录

> 变更日期：2026-06-23
> 变更原因：微信小程序通知能力无法满足核心需求
> 变更范围：平台、通知策略、定价、用户流程、技术架构

---

## 一、平台变更（核心）

### 变更：Phase 1 从微信小程序改为 iOS 原生 App

| 维度 | v1.1（小程序） | v2.0（iOS App） | 变更原因 |
|------|---------------|----------------|---------|
| 平台 | 微信小程序 | iOS 原生 App | 小程序订阅消息无法满足每日推送 |
| 推送机制 | 微信订阅消息（一次性授权=1次发送） | iOS 本地通知（无频次限制） | 小程序长期订阅不可申请 |
| 推送内快捷回复 | 不支持（需跳转小程序，2次点击） | Notification Actions（1次点击） | 减少操作成本 |
| 后台运行 | 不支持 | Background Tasks | 超时检测可靠性 |
| 联系人通知 | 免费=微信模板消息 / 付费=微信+SMS | 免费=SMS / 付费=SMS+语音电话 | 移除微信依赖 |
| 支付 | 微信支付 | Apple In-App Purchase | 平台切换 |
| 登录 | 微信一键登录 | 手机号 + 短信验证码登录 | 平台切换 |
| 联系人邀请 | 微信分享链接 | iOS 分享功能（iMessage/SMS/微信等） | 平台切换 |
| 小程序定位 | Phase 1 主体 | Phase 2 辅助入口（联系人看板/获客） | 降级 |

### 否决小程序的详细理由

微信小程序的订阅消息机制有三种类型：

1. **一次性订阅**：用户每次点击"允许"= 获得 1 次发送机会。发送后额度消耗。要每天推送就需要用户每天手动授权。
2. **长期订阅**：一次授权可长期发送。但仅限政务、医疗、金融等特定类目，"独居守护"不在开放范围内。
3. **服务通知**：用户最后一次操作后 7 天内可发送。超过 7 天静默。

我们的产品核心是**系统每天主动推送"今天还好吗？"**，要求：
- 无需用户每天手动操作就能收到推送
- 推送频次 = 每天 1 次，365 天不间断

这三种机制都无法满足。这不是技术实现问题，是微信平台策略限制。

## 三、通知策略变更

### 联系人通知通道

| 用户类型 | v1.1 | v2.0 |
|----------|------|------|
| 免费用户 | 微信模板消息（成本 0，但需联系人用过小程序） | **SMS 短信**（≤0.04元/条，100% 可达） |
| 付费用户 | 微信模板消息 + SMS 短信 | **SMS 短信 + 语音电话** |

### 变更原因

1. 移除微信模板消息后，免费用户必须改用 SMS，否则联系人无法收到通知
2. 微信模板消息有一个致命缺陷：联系人必须曾经使用过小程序才能收到——对于"被动接收通知"的联系人来说，这个前提经常不成立
3. SMS 虽然产生成本（~0.04元/条），但 100% 可达，不受平台限制
4. 付费用户增加语音电话通道，作为 SMS 的升级兜底

### 成本影响

| 场景 | v1.1 单次成本 | v2.0 单次成本 |
|------|-------------|-------------|
| 免费用户告警 | 0 元（微信模板） | ≤0.08 元（2轮SMS） |
| 付费用户告警 | ≤0.08 元（2轮SMS） | ≤0.36 元（2轮SMS + 2轮电话） |

## 四、用户流程变更

### 注册流程

| 步骤 | v1.1（小程序） | v2.0（iOS App） |
|------|---------------|----------------|
| 登录 | 微信一键授权 | 手机号 + 短信验证码 |
| 协议 | 独立页面 + 3条免责高亮 | 同（保持不变） |
| 基本信息 | 昵称 + 手机号 | 同 |
| 联系人 | 输入信息 + 微信邀请 | 输入信息 + 短信验证码 + 通讯录导入 |
| 提醒时间 | 设置时间窗口 | 同 |
| 通知授权 | 微信订阅消息授权 | **iOS 通知权限**（系统弹窗） |
| 完成 | 进入守护首页 | 同 |

### 每日提醒流程

| 环节 | v1.1 | v2.0 |
|------|------|------|
| 推送 | 微信订阅消息 | **iOS 本地通知**（设备本地调度，无需服务器） |
| 快捷回复 | 跳转小程序 → 点击按钮（2次点击） | **Notification Action 一键回复**（1次点击） |
| 关心式提醒 | 微信订阅消息 | **APNs 远程推送**（服务端触发） |
| 离线可靠性 | 不可靠（依赖微信平台） | **可靠**（本地通知不依赖网络） |

### 联系人通知流程

| 环节 | v1.1 | v2.0 |
|------|------|------|
| 通知方式 | 微信模板消息 / 微信+SMS | SMS / SMS+语音电话 |
| 通知内容 | 微信消息卡片 | SMS 文本 + 深链接 |
| 确认入口 | 微信消息内点击 → 小程序 | 深链接 → App（需安装）/ 短信回复 |
| 无 App 降级 | 联系人无需安装（小程序即用即走） | 联系人可直接电话联系，确认需下载 App |

### 子女邀请流程

| 环节 | v1.1 | v2.0 |
|------|------|------|
| 邀请方式 | 微信分享链接 | **iOS 分享功能**（iMessage/SMS/微信等全渠道） |
| 父母绑定 | 点击链接 → 微信授权 | 下载 App → 输入邀请码或点击深链接 |

## 六、技术架构变更

### v1.1 架构（小程序）

```
微信小程序 → 微信云开发（云函数 + 数据库 + 定时触发器）
通知：微信订阅消息 + 微信模板消息 + 阿里云短信 API
支付：微信支付
```

### v2.0 架构（iOS App）

```
iOS App (Swift/SwiftUI)
  ├── 本地通知 (UNUserNotificationCenter) — 每日提醒
  ├── 后台任务 (BGTaskScheduler) — 辅助超时检测
  ├── APNs 远程推送 — 关心式提醒、告警通知
  ├── iOS 分享 (UIActivityViewController) — 邀请联系人
  ├── Contacts Framework — 导入联系人
  └── CLLocationManager — 紧急定位

后端服务
  ├── 用户服务 — 注册、登录、配置
  ├── 守护引擎 — 超时检测（服务端定时器）、状态机
  ├── 通知服务 — APNs 推送、SMS 发送、语音电话
  ├── 支付服务 — Apple IAP 验证、订阅管理
  └── 数据库 — PostgreSQL / MySQL
```
