## 1. 概述

**今天还好**是一款面向独居人士的微信小程序，通过每日温暖的平安确认 + 超时自动通知紧急联系人，构建轻量化的独居安全守护网络。Phase 1 MVP 以最小成本验证"每日确认→超时通知→联系人接管"核心链路的真实需求和用户留存。

**核心价值主张**：不叫"签到"，不叫"打卡"——每天问一句"今天还好吗？"，让独居者用一次点击告诉关心自己的人"我没事"。

## 2. 背景与证据

### 2.1 市场背景

- 中国独居人口约 **1.2 亿**，且持续增长
- 独居者面临的核心焦虑："如果我出事了，多久才会有人知道？"
- 独居老人/慢性病患者的安全问题是家庭和社会痛点

### 2.2 竞品空白

| 竞品痛点 | 证据来源 |
|----------|----------|
| 通知渠道单一（仅邮件） | 死了么 App 评测 |
| 签到周期不合理（3天才触发） | 死了么 App 功能分析 |
| 无穿戴设备联动 | 全品类竞品扫描 |
| 缺乏主动预警 | 竞品功能对比 |
| 体验粗糙，缺乏信任感设计 | 独居签到/活着么 UI 分析 |

### 2.3 产品机会

现有竞品停留在"被动签到工具"层面，我们的差异化在于：
- **温暖交互**：以"今天还好"替代"签到"，降低心理负担
- **可靠通知**：免费用户也有短信通知联系人，核心守护链路不阉割
- **子女生态**：子女付费为父母开通，情感驱动付费
- **主动求助**：用户可主动发起紧急求助，不仅依赖被动检测
- **Phase 2 演进**：Apple Watch 心率联动 + 分级响应链（自动拨打120）

## 4. 用户与场景

### 4.1 目标用户

| 用户角色 | 描述 | 核心需求 |
|----------|------|----------|
| **独居者本人** | 独居青年/中年/老人 | 出事时有人知道；心理安全感；主动求助能力 |
| **紧急联系人** | 家人、朋友、邻居 | 被守护者出事时能被及时通知 |
| **子女** | 远在他城的子女 | 远程守护独居父母；主动查看安全状态 |

### 4.2 核心场景

**场景 A：独居者日常守护**
> 小李，28岁，独居。每天晚上 8 点收到微信消息"今天还好吗？"，点击通知跳转小程序，点一下"今天还好 ✓"，看到"收到，安心了"的反馈和"本月平安 19/22 天"的统计。整个过程不到 10 秒。

**场景 B：超时告警与联系人接管**
> 小李今天没回复。22:00 系统发关心式提醒"没收到你的回应，有点担心"，30 分钟宽限后仍未回复 → 状态变为"告警"，短信通知妈妈。妈妈打电话确认小李安全后，在小程序点"已联系，TA没事 ✓"，告警解除。

**场景 C：子女远程守护**
> 小张在深圳工作，妈妈在老家独居。小张为妈妈开通守护。妈妈不常回复，系统标记"未确认"（非告警），通知小张"妈妈今天还没回复"，小张打电话确认后在小程序代确认"TA没事"。代确认解除未确认状态，但不计入妈妈的日回复记录。小张付费解锁关怀看板，随时看妈妈的平安日历。

**场景 D：主动紧急求助**
> 小李在家摔倒但意识清醒，无法拨打电话。她打开小程序点击底部"紧急求助"，一键拨打 120 + 一键通知所有联系人"我需要帮助"。联系人收到特殊紧急短信。

**场景 E：暂停守护**
> 小李出差一周住酒店，在设置里选择"暂停守护 7 天"。暂停期间不收到提醒、不触发告警。7 天后自动恢复，收到"守护已恢复，今晚 8 点见"。

**场景 F：需要帮助**
> 联系人收到告警通知后联系不上被守护者，在小程序点"联系不上，需要帮助"，系统显示建议行动清单：拨打用户电话、拨打 120（提供用户紧急地址）、联系其他紧急联系人。Phase 1 不自动拨打 120，由联系人手动操作。

### 5.3 假设

1. 微信小程序订阅消息能满足每日提醒的频次需求（需验证微信平台策略）
2. 短信通知能有效触达紧急联系人（不依赖联系人是否使用过小程序）
3. 用户愿意为"多联系人 + 双通道通知 + 关怀看板"付费 9.9 元/月
4. 子女付费意愿强于独居者本人付费
5. 云端定时器（如微信云开发定时触发器）能满足超时检测的精度和可靠性
6. 微信邀请确认制比短信验证码有更高的联系人绑定率

### 5.4 依赖

| 依赖项 | 风险 | 缓解 |
|--------|------|------|
| 微信订阅消息授权率 | 用户可能拒绝授权，导致无法推送 | 引导流程强提示 + 小程序内备用入口 |
| 短信服务商稳定性 | 短信发送失败 | 选择主流服务商（如阿里云短信）+ 重试机制 |
| 微信支付审核 | 订阅付费需通过微信审核 | 提前准备资质；先用一次性付费过渡 |
| 联系人微信邀请确认率 | 联系人可能不打开邀请 | 手机号作为短信备用渠道，不绑定也能收到告警短信 |

## 6. 通知策略总表

| 场景 | 免费用户 | 付费用户 |
|------|---------|---------|
| 每日提醒 | 微信订阅消息 | 微信订阅消息 |
| 关心式提醒（宽限期） | 微信订阅消息 | 微信订阅消息 |
| 告警通知联系人 | **短信**（≤2条/事件，0.08元） | **短信 + 微信**双通道 |
| 第二轮联系人提醒 | ✗ | 短信（第2轮） |
| 紧急求助通知联系人 | 短信 | 短信 + 微信 |
| 子女"父母未回复"通知 | 微信订阅消息 | 微信订阅消息 |

**短信成本预估**：假设 15% 超时触发率，1000 DAU，日均告警事件 ≈ 150 次 × 2 条 × 0.04 元 = **12 元/天**，可控。

## 7. 用户流程

### 7.1 首次使用流程（4步精简版）

```
打开小程序
  │
  ▼
微信一键授权登录 + 同意协议（合并为一步）
  │
  ▼
设置紧急联系人（至少1人）
  ├─ 输入联系人姓名、手机号、关系
  ├─ 生成微信邀请消息 → 用户转发给联系人
  ├─ 联系人打开邀请 → 微信授权确认绑定
  └─ 或跳过绑定，仅保留手机号（短信通知仍可用）
  │
  ▼
设置提醒时间窗口 + 紧急地址
  ├─ 默认窗口：晚上 8-10 点
  └─ 紧急地址（选填）：用于紧急情况告知联系人和120
  │
  ▼
完成页（含订阅授权引导）
  ├─ 点击"开启提醒" → 触发订阅消息授权 → 进入首页
  └─ 跳过授权 → 进入首页（首页显示"未授权"提示卡片）
```

### 7.2 每日提醒流程

```
到达提醒时间窗口起始时间
  │
  ▼
推送微信订阅消息："今天还好吗？"
  │
  ├─ 用户点击通知 → 跳转小程序 → 点击"今天还好 ✓"
  │   └─ 显示温暖反馈："收到，安心了 ☺" + "本月平安 X/Y 天"
  │   └─ 预告："明天晚上 8 点再见"
  │
  └─ 用户未回复
      │
      ▼
  时间窗口结束 → 进入 30 分钟宽限
      │
      ▼
  推送关心式提醒："没收到你的回应，有点担心，看到消息回一下？"
      │
      ├─ 用户回复 → 状态正常，不触发联系人通知
      │
      └─ 仍未回复 → 30 分钟宽限结束
          │
          ▼
      ┌─ 用户有守护关系（被子女守护）→ 状态"未确认"→ 通知子女
      │   ├─ 子女代确认 → 解除未确认（不计入用户回复记录）
      │   └─ 子女未代确认 + 继续超时 → 升级为"告警"→ 通知紧急联系人
      │
      └─ 用户无守护关系 → 状态"告警"→ 短信通知紧急联系人
```

### 7.3 首页状态判定优先级

用户打开小程序时，按以下优先级决定显示哪个状态：

| 优先级 | 状态 | 判定条件 |
|--------|------|---------|
| 1（最高） | 守护已暂停 | 当前在暂停期内 |
| 2 | 告警中 | 当前有未解除的告警 |
| 3 | 宽限期 | 已过提醒窗口截止 + 30分钟宽限内 + 未回复 |
| 4 | 已回复 | 今天已回复 |
| 5 | 等待回复 | 当前在提醒时间窗口内 + 未回复 |
| 6（最低） | 非时段 | 当前不在提醒窗口内 + 无告警 + 未回复 |

### 7.4 联系人通知与接管流程

```
联系人收到短信通知
  │
  ▼
点击短信链接 → 进入小程序告警详情页
  │
  ├─ 点击"已联系，TA没事 ✓"
  │   └─ 告警解除，记录处理时间
  │   └─ 通知用户："你的联系人已确认你安全"
  │
  └─ 点击"联系不上，需要帮助"
      └─ 显示建议行动清单：
          ├─ 拨打用户电话 [一键拨号]
          ├─ 拨打 120 [一键拨号 + 显示用户紧急地址]
          └─ 联系其他紧急联系人 [列表 + 一键拨号]
      │
  10 分钟内所有联系人均未确认
      └─ 发送第二轮短信提醒给联系人（仅付费用户）
```

### 7.5 紧急求助流程（用户主动发起）

```
用户点击首页底部"紧急求助"
  │
  ▼
进入紧急求助页面
  ├─ 一键拨打 120（显示紧急地址）
  ├─ 一键通知所有联系人（发送"我需要帮助"短信）
  └─ 一键拨打联系人电话
  │
  ▼
联系人收到特殊短信："[用户] 主动发起紧急求助，请立即联系！"
```

### 7.6 子女端流程

```
子女打开小程序 → 选择"为家人开通守护"
  │
  ▼
填写父母信息（姓名、手机号）+ 设置提醒时间
  │
  ▼
生成邀请链接 → 通过微信发送给父母
  │
  ▼
父母点击链接 → 微信授权绑定 → 完成
  │
  ▼
子女自动成为父母的第一紧急联系人
  ├─ 代设提醒时间窗口
  └─ 管理其他联系人
  │
  ▼
日常：父母未回复 → 标记"未确认"（非告警）
  ├─ 通知子女："妈妈今天还没回复"
  └─ 子女可代确认"TA没事"（解除未确认，但不计入父母回复记录）
  │

关怀看板（付费解锁）：
```

### 7.7 暂停守护流程

```
用户进入设置 → 点击"暂停守护"
  │
  ▼
选择暂停时长：1天 / 3天 / 7天 / 自定义
  │
  ▼

  ├─ 首页变为"守护已暂停"状态
  ├─ 暂停期间：不发送提醒、不触发超时、不通知联系人
  └─ 关怀看板标记暂停天为"暂停"而非"未回复"
  │
  ▼

  └─ 推送通知："守护已恢复，今晚 X 点见"
  │
  用户也可随时点击"立即恢复守护"手动恢复
```

## 8. 功能需求

### R1：用户注册与身份绑定（REQ-001）

**用户故事**：作为独居者，我希望用微信一键登录并快速完成设置，以便尽快开始使用守护服务。

**功能行为**：
- 微信一键登录，获取 OpenID + 头像 + 昵称
- 昵称和手机号从微信 profile 自动获取，用户可修改
- **4步精简引导**：登录+协议 → 联系人 → 提醒时间+地址 → 完成（含授权）
- 免责协议在登录页底部以链接形式展示，不单独成页
- 订阅消息授权移到完成页，作为"开启提醒"按钮的一部分
- 未完成引导的用户下次打开时从断点继续

**数据/字段**：
- 用户表：openID, 昵称, 手机号, 紧急地址（可选）, 注册时间, 引导完成状态, 订阅授权状态, 付费状态, 守护暂停到期时间（可选）

**边界情况**：
- 用户中途退出引导 → 下次打开从断点继续
- 手机号已被其他账号使用 → 提示"该手机号已绑定其他账号"

**验收标准**：
- [ ] 微信授权后 3 步内完成引导（联系人 → 提醒设置 → 完成）
- [ ] 未授权订阅消息时，完成页显示"未授权"提示卡片，可重新授权
- [ ] 完成后自动进入守护首页
- [ ] 中途退出后再次打开可从断点继续

---

### R2：紧急联系人管理（REQ-002）

**用户故事**：作为独居者，我希望设置至少一个紧急联系人，确保出事时有人被通知。

**功能行为**：
- 免费用户：最多 1 个紧急联系人
- 付费用户：最多 5 个紧急联系人 + 优先级排序
- 联系人信息字段：姓名、手机号、与用户关系、微信 OpenID（通过邀请绑定）
- **微信邀请确认制**：
  1. 用户填写联系人信息

### R4：超时检测与联系人通知（REQ-004）

**边界情况**：

- 短信发送失败 → 重试 1 次，仍失败则记录日志
- 用户连续 3 天超时 → 在小程序内弹出"提醒时间调整建议"

**验收标准**：
- [ ] 提醒截止后 30 分钟宽限期内回复 → 不触发联系人通知
- [ ] 宽限期间的关心式提醒语气温暖，非系统告警风格
- [ ] 免费用户联系人收到短信通知（≤2条/事件）
- [ ] 付费用户联系人同时收到短信 + 微信通知

- [ ] 通知内容包含用户昵称、最后回复时间、建议行动
- [ ] 联系人收到通知后可一键进入小程序查看状态
- [ ] 子女端可代确认"TA没事"，解除未确认状态
- [ ] 子女代确认不计入用户本人回复记录
- [ ] "未确认"和"告警"状态在 UI 上有明确视觉区分

**验收标准**：
- [ ] 设置页有"暂停守护"入口
- [ ] 可选 1/3/7 天或自定义暂停时长
- [ ] 暂停期间不发送提醒、不触发告警、不通知联系人
- [ ] 暂停到期后自动恢复并推送通知
- [ ] 用户可随时手动恢复守护
- [ ] 关怀看板中暂停天标记为"暂停"而非"未回复"

## 9. 数据与权限要求

### 9.1 数据模型概览

```
用户 (User)
├── openID (PK)
├── 昵称
├── 手机号
├── 紧急地址（可选）
├── 提醒窗口（开始/结束）
├── 订阅状态
├── 订阅授权状态
├── 引导完成状态
├── 守护暂停到期时间（可选）
└── 付费状态

联系人 (Contact)
├── ID (PK)
├── 用户ID (FK)
├── 姓名、手机号、关系
├── 微信OpenID
├── 绑定方式（微信邀请确认 / 仅手机号）
├── 绑定状态（已绑定 / 待确认 / 仅手机号）
├── 优先级
└── 创建时间

回复记录 (ReplyLog)
├── ID (PK)
├── 用户ID (FK)
├── 回复时间
├── 回复方式（通知跳转/小程序内）
└── 回复类型（本人回复 / 子女代确认）

守护状态 (GuardStatus)
├── 用户ID (PK)
├── 当前状态（正常/未确认/告警/暂停）
├── 最后回复时间
└── 告警触发时间

通知记录 (NotificationLog)
├── ID (PK)
├── 联系人ID (FK)
├── 通知时间、渠道（短信/微信）、轮次
└── 送达状态

守护关系 (GuardRelation)
├── 子女用户ID
├── 父母用户ID
└── 关系类型

告警处理记录 (AlertAction)
├── 告警ID (FK)
├── 联系人ID (FK)
├── 操作（已联系/需要帮助）
├── 操作时间
├── 告警来源（超时触发 / 用户主动求助）
└── 代确认标记（是否由子女代确认）

协议同意记录 (AgreementLog)
├── 用户ID (FK)
├── 协议版本
└── 同意时间

订阅记录 (Subscription)
├── 用户ID (FK)
├── 订阅状态、开始时间、到期时间
├── 自动续费
└── 付费者（本人/子女）
```

### 9.2 权限矩阵

| 角色 | 可查看数据 | 可操作 |
|------|-----------|--------|
| 独居者本人 | 自己的回复记录、联系人、守护状态、告警历史 | 回复、紧急求助、管理联系人、设置提醒时间、暂停/恢复守护、删除数据 |
| 紧急联系人（免费版） | 当前守护状态 + 最后回复时间 | 确认/接管告警 |
| 紧急联系人（付费版） | 守护状态 + 7天平安日历 + 月度统计 | 确认/接管告警 |
| 子女 | 同付费版联系人 + 代确认权限 | 代确认、管理父母提醒时间和联系人、为父母付费 |

### 9.3 隐私策略

- **最小数据原则**：仅采集守护功能所必需的数据
- **紧急地址**：用户选填，仅在告警和紧急求助时展示给联系人
- **短信记录**：仅保留发送日志（时间、接收人），不保留短信内容
- **数据可删除**：用户可随时删除所有数据，输入"确认删除"后不可恢复
- **子女权限隔离**：子女无法查看/修改父母的核心隐私数据（手机号、地址）

## 10. 验收标准汇总

### 10.1 主流程（Happy Path）

- [ ] 新用户 3 步内完成引导并进入守护首页
- [ ] 在设定时间窗口内收到"今天还好吗？"推送（误差 ±15 分钟）
- [ ] 点击通知跳转小程序后，1 次点击完成回复，收到温暖反馈
- [ ] 超时 30 分钟宽限后，联系人收到短信通知
- [ ] 联系人点击"已联系，TA没事"→ 告警立即解除
- [ ] 子女为父母开通守护 → 父母一键绑定 → 子女可查看状态
- [ ] 用户点击"紧急求助" → 可一键拨打 120 + 通知联系人

### 10.2 异常路径

- [ ] 用户未授权订阅消息 → 小程序内引导授权
- [ ] 推送发送失败 → 记录日志，不触发误报告警
- [ ] 联系人 10 分钟未确认 → 第二轮短信提醒（付费用户）
- [ ] 联系人点击"需要帮助" → 显示行动建议清单（含 120 拨打指引 + 紧急地址）
- [ ] 联系人未绑定微信 → 仍可通过短信收到通知

### 10.3 数据一致性

- [ ] 用户回复后，守护状态 ≤ 1 分钟更新
- [ ] 告警解除后，所有联系人看到已解除状态
- [ ] 联系人变更后的下次提醒中提示用户确认
- [ ] 删除数据操作后，所有关联记录同步清除
- [ ] 子女代确认不计入用户本人回复记录
- [ ] 暂停守护期间，关怀看板标记为"暂停"而非"未回复"
